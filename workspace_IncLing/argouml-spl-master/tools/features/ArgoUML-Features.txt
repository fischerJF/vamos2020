PREMISSAS:
	- Não haverá tratamento nas classes de testes (fonte gerado não terá jUnits);
	- Marcação da granularidade do código da feature:
			//@#$LPS-FEATURE:GranularityType:TIPOGRANULARIDADE, no qual FEATURE deverá ser substituído por uma das 
				features (COGNITIVE, ACTIVITYDIAGRAM, STATEDIAGRAM, LOGGING) e TIPOGRANULARIDADE deverá ser substituído pelos tipos definidos abaixo.
				
		- Tipos de granularidade:
			- Package (Pacote inteiro - todas as classes. Se houver uma classe do pacote que não deve ser marcada, todas devem ser marcadas como Class)
			- Import (Se houver mais de um import em sequência, contar uma vez - Bloco)
			- Class
			- ClassSignature (Quando se altera um "extend", "implements" em uma classe)
			- Interface (Uma interface inteira)
			- InterfaceMethod (Alteração de uma métodos de interface)
			- Method (Assinatura + corpo)
			- MethodBody (Apenas corpo do método)
			- MethodSignature (Quando altera os arqumentos do métodos)
			- MethodCall (Quando se altera os arqumentos de uma chamada de método)
			- Field (Campo de uma classe. Deverá contar todos os campos)	
			- Variable (Variável dentro de um método)	
			- Expression (Quando se altera uma expressão booleana (if ou return))
			- Command (Comandos de atribuição, chamada de método, comandos for, if, else if, else, while - Bloco)
					Para um bloco de comandos, contar apenas uma vez.
					Por ex., para um if, contar somente o if, ou vários comandos consecutivos, contar apenas um comando
			- Enumeration (Alteração em elementos de uma enumeração)
			- StaticInitialization (Inicialização estática de vetores - Bloco)
		
	- Marcar localização do código da Feature:
			//@#$LPS-FEATURE:Localization:TIPOLOCALIZAÇÃO, no qual FEATURE deverá ser substituído por uma das 
				features (COGNITIVE, ACTIVITYDIAGRAM, STATEDIAGRAM, LOGGING) e TIPOLOCALIZAÇÃO deverá ser substituído pelos tipos definidos abaixo.
		
		Tipos de localização (considerar o bloco):
			- StartMethod (Início de método)
			- EndMethod (Fim de método)
			- EntireMethod (Quando a marcação ocupa o método todo)
			- BeforeReturn (Antes de um return - Return não está marcado, mas o código imediatamente anterior está, não apenas sintaticamente antes, mas logicamente)
			- StartStaticBlock (Início de um bloco estático)
			- EndStaticBlock (Final de um bloco estático)
			- EntireStaticBlock (Bloco estático inteiro)
			- InsideStaticBlock (Dentro de um bloco estático)
			- NestedCommand (Um comando aninhado marcado dentro de um comando não aninhado. Exceto para blocos try/catch)
					Ex.: 	Comando	-> Este comando não está marcado
								Comando -> Este comando está marcado, contar como NestedCommand
